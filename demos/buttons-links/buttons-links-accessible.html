Show fixes and notes

Fixes

  • Fake button supplied with role="button" attribute (in addition to tabindex="0") and keydown listener for enter and space keys (new technique).
  • Native button element used.
  • Visually hidden text alternative provided
  • Icon element hidden reliably from screenreaders using a custom element with aria-hidden="true" role="presentation"
  • Native focus outline preserved (G165) and :hover styles applied to :focus, too.

Notes

  • Corresponding success criterion for keyboard navigation: 2.1.1 Keyboard.
  • The preferred solution regarding buttons is to use a real button or link, see H91. Further reading: Just use a (native HTML) button by Steve Faulkner, Links are not buttons. Neither are DIVs and SPANs by Karl Groves.
  • Corresponding success criterion for focus styles: 2.4.7 Focus Visible.
  • Alternative approaches regarding outlines: Don't Lose Your :focus by Patrick H. Lauke and How to remove CSS outlines in an accessible manner? by Steve Faulkner.
  • JavaScript solution to detect input mode: What input

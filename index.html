Demos

«How to make your frontend accessible»

Frontend Conference Zürich 2013, Werner Hänggi & Thomas Jaggi

1. Tables

  • Demo 1.1: Inaccessible table
    • Cells not connected to their headers.
    • Caption not connected to table.
  • Demo 1.2: Accessible table
    • th with scope attribute used.
    • caption used.

2. Form markup and validation

  • Demo 2.1: Inaccessible form
    • label tags omitted or hidden.
    • Required fields not properly marked.
    • Unnoticed insertion of error message.
    • No connection of error messages to invalid fields.
  • Demo 2.2: Accessible form
    • label tags used.
    • Required fields indicated with text.
    • Error messages inserted into label tags.
    • First invalid field focused.
  • Demo 2.3: Accessible form validation using ARIA
    • aria-required used to mark required field.
    • aria-invalid used to dynamically mark invalid field.
    • aria-labelledby used to connect error messages to field.
  • Demo 2.4: HTML5 form validation
    • required attribute used to mark required fields.

3. Element focus and keyboard navigation

  • Demo 3.1: Inaccessible focus handling and keyboard navigation
    • "outline:none" used to remove focus indication.
    • Fake buttons used, not triggered by keyboard and partly unfocusable.
  • Demo 3.2: Accessible focus handling and keyboard navigation
    • Native focus outline preserved.
    • Fake buttons supplied with role="button" attribute and keydown listener for enter and space keys the (preferred solution is always a real button or link!).
